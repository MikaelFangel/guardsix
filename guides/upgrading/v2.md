# Upgrading to v2

v2 is a full redesign of the library. This guide covers every breaking change
and how to migrate your code.

## Dependency

```elixir
# v1
{:logpoint_api, "~> 1.0.0"}

# v2
{:logpoint_api, "~> 2.0.0"}
```

The HTTP client changed from HTTPoison to Req, and Joken was added for
JWT-authenticated endpoints (alert rules, repos).

## Client setup

The plain credentials map is replaced by a `Client` struct built via the
facade.

```elixir
# v1
credentials = %{
  ip: "192.168.1.10",
  username: "admin",
  secret_key: "mysecret",
  verify_ssl: false
}

# v2
client = LogpointApi.client("https://192.168.1.10", "admin", "mysecret", ssl_verify: false)
```

Key differences:

- `ip` is now a full URL (`base_url`)
- `verify_ssl` is renamed to `ssl_verify`
- SSL verification defaults to `true` (was `false` in v1)

## Search

<code>LogpointApi.Query</code> is replaced by `LogpointApi.search_params/4,5`.
The auto-polling <code>run_search/2</code> is removed. See the
[Polling for Search Results](polling-search-results.md) guide for how to
implement polling yourself.

```elixir
# v1
query = %LogpointApi.Query{
  query: "user=*",
  time_range: [1_714_986_600, 1_715_031_000],
  limit: 100,
  repos: ["127.0.0.1:5504"]
}
{:ok, result} = LogpointApi.run_search(credentials, query)

# v2
alias LogpointApi.Core.Search

query = LogpointApi.search_params("user=*", 1_714_986_600, 1_715_031_000, 100, ["127.0.0.1:5504"])
{:ok, %{"search_id" => id}} = Search.get_id(client, query)
{:ok, result} = Search.get_result(client, id)
```

| v1 | v2 |
|---|---|
| <code>LogpointApi.run_search/2</code> | Removed |
| <code>LogpointApi.get_search_id/2</code> | `Search.get_id/2` |
| <code>LogpointApi.get_search_result/2</code> | `Search.get_result/2` |
| <code>LogpointApi.user_preference/1</code> | `Search.user_preference/1` |
| <code>LogpointApi.logpoint_repos/1</code> | `Search.logpoint_repos/1` |
| <code>LogpointApi.devices/1</code> | `Search.devices/1` |
| <code>LogpointApi.livesearches/1</code> | `Search.livesearches/1` |

## Incidents

All incident functions moved to `LogpointApi.Core.Incident`.

```elixir
# v1
{:ok, incidents} = LogpointApi.incidents(credentials, start_time, end_time)
{:ok, _} = LogpointApi.resolve_incidents(credentials, ["id1"])
{:ok, _} = LogpointApi.add_comments(credentials, %{"id1" => ["note"]})

# v2
alias LogpointApi.Core.Incident

{:ok, incidents} = Incident.list(client, start_time, end_time)
{:ok, _} = Incident.resolve(client, ["id1"])

comment = LogpointApi.comment("id1", "note")
{:ok, _} = Incident.add_comments(client, [comment])
```

| v1 | v2 |
|---|---|
| <code>LogpointApi.incidents/3</code> | `Incident.list/3` |
| <code>LogpointApi.incident_states/3</code> | `Incident.list_states/3` |
| <code>LogpointApi.incident/3</code> | `Incident.get/3` |
| <code>LogpointApi.add_comments/2</code> | `Incident.add_comments/2` |
| <code>LogpointApi.assign_incidents/3</code> | `Incident.assign/3` |
| <code>LogpointApi.resolve_incidents/2</code> | `Incident.resolve/2` |
| <code>LogpointApi.close_incidents/2</code> | `Incident.close/2` |
| <code>LogpointApi.reopen_incidents/2</code> | `Incident.reopen/2` |
| <code>LogpointApi.users/1</code> | `Incident.get_users/1` |

New in v2: `list/4` and `list_states/4` accept an optional filters map.

## Removed modules

| Removed | v2 replacement |
|---|---|
| <code>LogpointApi.Query</code> | `LogpointApi.search_params/4,5` |
| <code>LogpointApi.Incident</code> | `Incident.get/3` |
| <code>LogpointApi.TimeRange</code> | Inline args |
| <code>LogpointApi.IncidentComment</code> | `LogpointApi.comment/2` |
| <code>LogpointApi.IncidentCommentData</code> | Pass `[Comment.t()]` directly |
| <code>LogpointApi.IncidentIDs</code> | Pass `[String.t()]` directly |

## New in v2

These domains are entirely new and have no v1 equivalent:

- **Alert Rules** &mdash; `LogpointApi.Core.AlertRule` with builder pattern via `LogpointApi.rule/1`
- **Email Notifications** &mdash; builder via `LogpointApi.email_notification/2`
- **HTTP Notifications** &mdash; builder via `LogpointApi.http_notification/3`
- **Logpoint Repos** &mdash; `LogpointApi.Core.LogpointRepo`
- **User-Defined Lists** &mdash; `LogpointApi.Core.UserDefinedList`
- **Incident filtering** &mdash; optional filters map on `list/4` and `list_states/4`

See the `LogpointApi` module docs for full usage examples.
